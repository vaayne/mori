Mori · Sidebar Redesign · v1

从「文件树」到「注意力收件箱」

核心判断：用户打开 sidebar 不是为了浏览项目，而是想知道「哪个 agent 在等我」。 重设计把等待中的 agent 提到第一屏、可直接内联回复，状态只用一套颜色点语言表达， 无活动的项目自动收纳成一行。下面是可交互原型 — 试试点击 Needs You 卡片回复、切换过滤器、折叠成 Rail。

设计决策 — 相对现状改了什么、为什么

01

Needs You 收件箱，替代 Agents 列表

原版 Agents 区只告诉你「claude 在等」，你还得切过去看它问什么。新版直接展示 agent 最后一行输出（来自现有的 pane output preview 能力），并支持内联回复（复用 QuickReplyField）— 多数回复是 "yes" / "go on"，根本不值得切换上下文。

02

一套状态语言：颜色点，不是文字 pill

原版同一个 Waiting 状态出现三次：summary strip、Agents 区、项目行黄色 pill。新版统一为点的颜色：琥珀呼吸＝等待，绿色＝运行，空心＝闲置，红＝错误。等待时长（4m）比 "Waiting" 这个词信息量大得多。

03

Summary strip 变成过滤器

原版顶部的 "4 waiting · 1 running · 13 trees" 只能看不能点。新版做成 segmented filter — 点 ● 4 直接过滤出所有等待项。可看的数据不如可操作的数据。

04

闲置项目自动收纳

截图里 13 个项目，7 个完全没有活动，却各占一张全尺寸卡片。新版把无活动且未 pin 的项目折叠成一行 chip 集群，点击即唤醒。Sidebar 高度从 ~1400px 压缩到一屏以内。

05

Rail 模式重做：一个 tile 说清一切

原版 rail 每个项目两个圆圈（项目字母 + worktree 字母），一排 "M" 没有任何信息量。新版每项目一个 tile：状态环颜色＝最高优先级状态，角标数字＝等待数，hover 弹出 worktree 列表。

06

去掉卡片化，回归层级

原版每个项目组一个带边框的卡片盒子，边框比内容还显眼。新版项目行无边框，worktree 用左侧细线挂接，选中态保留现有的 accent bar + 渐变。视觉重量留给真正需要注意的东西。所有改动都映射到现有模型（AgentState / AlertState / isFavorite），无需动数据层。
